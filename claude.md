FLEXIBLE CODE LOCKING SYSTEM
This is the most important architectural decision. Do NOT use a top_code and bottom_code split. That is rigid and does not scale. Instead, store the full template as a single string called template_code, and store editable zones as a JSON array called editable_ranges like this: [{"startLine": 4, "endLine": 5}, {"startLine": 9, "endLine": 11}]. Everything NOT in editable_ranges is locked. This supports any number of locked and editable regions anywhere in the code, which is how real interview platforms work.

AST CONSTRAINT ENGINE
This is the hero feature. Use proper AST parsing for all 5 languages. No keyword scanning. For Python use the built-in ast module, walk the syntax tree, detect ast.While for while loops, ast.For for for loops, ast.If for conditionals, ast.Match for switch, ast.FunctionDef plus ast.Call for recursion detection (function calling itself by name), ast.ClassDef for classes, ClassDef.bases for inheritance, ast.Try for exception handling, ast.List ast.Dict ast.Set and deque calls for data structures. For Java use JavaParser, detect WhileStmt, ForStmt, DoStmt, IfStmt, SwitchStmt, MethodDeclaration, recursive method calls, ClassOrInterfaceDeclaration, superclass for inheritance, TryStmt, CatchClause, and collection types like ArrayList LinkedList Stack Queue. For C use pycparser, detect while loops, for loops, do-while, if statements, switch statements, function declarations, recursive calls, structs and arrays for data structures. C has no native exception handling so skip that constraint for C and add a comment explaining why. For C++ use libclang Python bindings, detect CursorKind.WHILE_STMT, FOR_STMT, DO_STMT, IF_STMT, SWITCH_STMT, FUNCTION_DECL, recursive calls, CLASS_DECL, base class for inheritance, CXX_TRY_STMT, and STL types like vector list stack queue. For JavaScript use acorn or esprima, detect WhileStatement, ForStatement, DoWhileStatement, IfStatement, SwitchStatement, FunctionDeclaration, FunctionExpression, ArrowFunctionExpression, recursion, ClassDeclaration, class heritage for inheritance, TryStatement, CatchClause, and array/Map/Set usage.
The engine must support validating all of these constructs: while loops, for loops, do-while loops, if/else, switch, functions, recursion, classes, inheritance, polymorphism, exception handling, arrays, linked lists, stacks, and queues.
After getting violations from the analyzer, filter them so that only violations falling within editable_ranges are surfaced to the user. Violations in locked zones are ignored since the editor already prevents editing there. Line numbers from the analyzer and editable_ranges must use the same indexing — verify this explicitly.
